# 📑 Resumo de Estudos: Sistemas Operacionais e Git

## 1. Introdução aos Sistemas Operacionais (SO)

O **Sistema Operacional** é o software mestre que gerencia os recursos do computador, servindo de ponte entre o hardware, os aplicativos e o usuário.

### Funções Principais:
* **Gestão de Processos:** Coordena a execução de programas pela CPU.
* **Gestão de Memória:** Organiza como os dados são carregados e lidos.
* **Controle de Dispositivos:** Gerencia periféricos e hardware interno.
* **Segurança:** Mantém a estabilidade e proteção do sistema contra falhas.

---

## 2. Classificação dos Sistemas Operacionais

### 🏢 Sistemas de Grande Porte e Rede
* **Mainframes:** Projetados para processar volumes massivos de dados com altíssima confiabilidade (ex: Bancos).
* **Servidores:** Focados em prover serviços para múltiplos usuários através de redes (ex: Windows Server, Linux).

### 💻 Uso Pessoal e Mobilidade
* **Computadores Pessoais:** Sistemas para desktops e notebooks com foco em interface gráfica (ex: Windows, macOS).
* **Portáteis (Mobile):** Otimizados para economia de energia e uso de sensores (ex: Android, iOS).

### ⚙️ Sistemas Especializados
* **Multiprocessadores:** Utilizam múltiplos núcleos de CPU para alto desempenho.
* **Embarcados:** Sistemas dedicados em dispositivos como Smart TVs e micro-ondas.
* **Nós Sensores:** Pequenos dispositivos conectados em rede para monitoramento ambiental.
* **Tempo Real (Real-Time):** Sistemas onde o tempo de resposta é vital. Podem ser **Hard** (rígidos) ou **Soft** (flexíveis).
* **Cartões Inteligentes:** Sistemas em chips com foco em segurança e criptografia.

---

## 🛠️ Controle de Versão com Git

O **Git** é uma ferramenta indispensável para rastrear mudanças em projetos de software e permitir o trabalho colaborativo.

### Fluxo de Trabalho e Instalação:
1. **Instalação:** Via site oficial, verificando com `git --version` no terminal.
2. **Inicialização:** Comando `git init` na pasta do seu projeto.
3. **Commit:** Registrar uma "foto" do projeto com o comando de commit.
4. **GitHub:** Publicar o código em repositórios remotos para backup e colaboração.

### ✅ Boas Práticas:
* Realizar **commits pequenos** e frequentes.
* Escrever **mensagens claras** para descrever as mudanças.
* Utilizar **branches** para desenvolver novas funcionalidades separadamente.
* Sempre realizar **testes** antes de integrar (merge) as alterações.
